TECHNICAL DETAILS
================================================================================

Image Path Resolution:
   NUI Base: nui://my_dealership/html/
   Image Path: images/{model}.jpg
   Full URL: nui://my_dealership/html/images/rhapsody.jpg

Loading Sequence:
   1. User selects vehicle
   2. renderDetailsPanel() creates HTML with image element
   3. Browser attempts to load: images/{model}.jpg
   4. Success → fade in | Error → show placeholder

FiveM NUI Notes:
   - Files must be in fxmanifest.lua files{} declaration
   - Wildcard *.jpg includes all JPG files
   - Resource restart required after adding images
   - NUI uses Chromium Embedded Framework

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Images don't appear
Solution:
   - Check files exist: ls my_dealership/html/images/
   - Verify filenames match model names (lowercase)
   - Ensure .jpg extension
   - Restart resource: restart my_dealership

Problem: Placeholder always shows
Solution:
   - Image file doesn't exist or wrong filename
   - Check browser console (F8) for 404 errors
   - Verify file extension is .jpg not .jpeg

Problem: Images load slowly
Solution:
   - Compress images (reduce file size)
   - Use 80-85% JPG quality
   - Resize to ~800x450px before adding
